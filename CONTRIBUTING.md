# Contributing to StudentOS

Thank you for your interest in improving StudentOS.

## How to contribute

1. Fork the repository.
2. Create a feature branch.
3. Make your changes with clear, maintainable code.
4. Run the project and confirm it builds.
5. Open a pull request with a clear description.

## Code style

- Keep Kotlin code readable and idiomatic.
- Prefer small, focused composables.
- Avoid unnecessary duplication.
- Use descriptive names for screens, functions, and variables.

## Pull request checklist

- The app builds successfully.
- The change is documented if needed.
- The PR description explains the purpose and result.
